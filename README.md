# 푸린네 태교 다이어리 🌙

누오(남편) · 푸린(아내) · 푸푸린(아기)의 하루하루를 기록하는, 게임 같은 태교 응원 페이지입니다.
매일 밤 페이지를 열고 **"오늘도 무사히 하루 클리어"** 를 누르면 그날이 기록되고, 아기 푸푸린이 조금씩 자랍니다. 주수 마일스톤에 도달하면 특별한 진화 이벤트가 열려요.

## 파일 구조

| 파일 | 하는 일 |
|---|---|
| `index.html` | 화면 구조 |
| `style.css` | 밤 테마 디자인 |
| `app.js` | 게임 로직 (D-day·주수·스트릭·마일스톤) |
| `config.js` | **예정일/주수/마일스톤 설정** — 여기 숫자만 바꾸면 됩니다 |
| `messages.js` | **누오의 편지 + 시스템 응원 문구** — 날짜별 편지와 게임 응원 문구 |

## 화면 기능

- **홈** — 임신 주수·D-day, "오늘도 무사히 하루 클리어" 버튼, **매일 자동으로 바뀌는 시스템 응원**(게임 말투), 누오의 오늘 편지
- **도감** — 함께한 날/연속 기록, 주수 마일스톤 뱃지
- **편지함(우리 이야기)** — **엄마·아빠가 각자 메시지를 남기고**, 아래에서 지난 메시지와 누오의 편지를 다시 읽을 수 있어요

### 엄마·아빠 메시지 저장 방식
- `firebase-config.js`를 **비워두면**: 메시지가 **그 기기에만** 저장돼요 (같은 폰에서 번갈아 쓰는 추억 기록용).
- `firebase-config.js`를 **채우면**: 아빠 폰에서 쓴 메시지가 **엄마 폰에도 실시간으로** 뜹니다. (아래 참고)

## Firebase 연결하기 (다른 기기 간 메시지 공유)

무료로 5분이면 됩니다.

1. [console.firebase.google.com](https://console.firebase.google.com) 접속 → **프로젝트 만들기** (이름 아무거나, 애널리틱스는 꺼도 됨)
2. 왼쪽 **빌드 → Firestore Database → 데이터베이스 만들기** → 위치 선택 → 우선 "테스트 모드"로 시작
3. 프로젝트 개요 옆 **⚙️ → 프로젝트 설정 → 내 앱 → 웹(</>) 앱 추가** → 앱 등록하면 `firebaseConfig` 값이 나옵니다
4. 그 값(apiKey, authDomain, projectId …)을 `firebase-config.js`에 그대로 붙여넣기
5. 다시 배포(push)하면 끝. 이제 두 폰에서 같은 주소를 열면 메시지가 공유돼요.

### Firestore 규칙
테스트 모드는 30일 뒤 잠깁니다. **Firestore → 규칙**에 아래를 넣으세요. (아래는 로그인 없이 열어두는 규칙 — 앱 암호 잠금과 함께 쓰는 방식입니다.)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /notes/{doc} {
      allow read, write: if true;
    }
  }
}
```

## 공유 암호 잠금 (권장)

위 규칙은 DB를 열어두기 때문에, **앱에 두 분만 아는 암호**를 걸어 다른 사람이 페이지를 열지 못하게 합니다.

1. 저장소의 **`hash.html`** 을 브라우저로 엽니다
2. 두 분이 쓸 암호(예: `사랑해푸린2026`)를 입력 → **지문 만들기** → 64자리 지문이 나옵니다
3. 그 지문을 **`config.js`의 `gateHash`** 에 붙여넣기
4. 배포하면, 앱을 열 때 암호를 물어봅니다. 한 번 맞히면 그 기기에선 다시 안 물어봐요.

> **왜 지문(해시)인가**: 정적 사이트는 암호 원문을 숨길 수 없어서, 되돌릴 수 없는 지문만 코드에 넣습니다. 코드를 봐도 원문은 안 보여요. (암호는 좀 길게 잡으세요.)
>
> **한계(꼭 아세요)**: 이 잠금은 "웹페이지 열기"를 막아줍니다. 하지만 DB 규칙이 열려 있으면, 공개 저장소와 Firebase 설정을 찾은 기술적인 사람은 앱을 거치지 않고 DB를 볼 여지가 있어요. 가족 추억 기록 + 주소 비공개면 실용적으로 충분하지만, **완전한 차단이 필요하면 구글 로그인(두 분 계정만 허용) 방식으로 업그레이드**하세요.

## 편지 채우는 법 (가장 중요!)

`messages.js` 를 열면 세 부분이 있어요:

```js
daily: {
  "2026-07-19": "여기에 그날 하고 싶은 말을 적어요",
  "2026-07-20": "...",
}
```
- `daily` — **날짜별 편지.** 하루에 하나씩, 남편분 진짜 목소리로 채워주세요. (초안 2주치가 들어 있어요 — 마음껏 고치세요.)
- `milestones` — 24/28/32/34/37주 도달 시 뜨는 특별 편지.
- `fallback` — 그날 편지를 안 적은 날 랜덤으로 보여줄 문구. 편지를 매일 못 채워도 페이지가 비지 않아요.

> 팁: 여유 있을 때 미리 여러 날치를 적어두면, 아내분이 매일 밤 "오늘의 새 편지"를 받게 됩니다.

## 설정 바꾸기

`config.js` 의 `dueDate`(예정일)만 맞으면 주수·D-day·마일스톤 날짜가 전부 자동 계산됩니다. 예정일이 바뀌면 이 값만 수정하세요.

## GitHub Pages 에 올리는 법

1. GitHub 에서 새 저장소(repository)를 만듭니다. 예: `purin-diary`
2. 이 폴더의 파일 전부를 저장소에 올립니다.
   ```bash
   git init
   git add .
   git commit -m "푸린네 태교 다이어리"
   git branch -M main
   git remote add origin https://github.com/<내아이디>/purin-diary.git
   git push -u origin main
   ```
3. 저장소 → **Settings → Pages** → Source 를 `Deploy from a branch`, Branch 를 `main` / `root` 로 지정 → Save
4. 1~2분 뒤 `https://<내아이디>.github.io/purin-diary/` 로 접속됩니다.
5. 아내분 폰에서 그 주소를 열고 **홈 화면에 추가**하면 앱처럼 쓸 수 있어요.

### 비공개로 쓰고 싶다면
GitHub Pages 는 무료 요금제에선 공개됩니다. 남에게 보이는 게 싫다면 주소를 아무에게도 공유하지 않으면 사실상 노출되지 않지만, 완전한 비공개가 필요하면 GitHub Pro(비공개 Pages) 또는 Netlify/Vercel 의 비밀번호 보호를 쓰세요.

## 저장 방식

기록은 서버가 아니라 **접속한 브라우저(localStorage)** 에 저장됩니다. 즉 아내분이 늘 같은 폰·같은 브라우저로 열어야 기록이 이어져요. (기기를 바꾸면 기록이 새로 시작됩니다.)
